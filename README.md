# 薪資差異分析 · 月對月

純前端網頁工具，回答老闆最常問的一句：**「為什麼這個月人數少了，總額卻增加？」**

直接讀公司的 `薪资.xlsx`（每月一個分頁），在瀏覽器裡算，**資料不上傳任何伺服器**。

## 用法
1. 用瀏覽器打開 `index.html`（或部署到 GitHub Pages / 任何靜態空間）
2. 把 `薪资.xlsx` 拖進去
3. 選「本月 / 上月」兩個分頁（匯率會自動帶入，可手改）
4. 按「分析」

## 它算什麼
把總額差異拆成一條會對帳的瀑布：

```
上月總額 → +匯率效應 → +實發RMB差 → +PESO補貼差 → +PESO扣款差 → 本月總額
```

- **匯率效應** = 本月實發RMB ×（本月匯率 − 上月匯率）。把匯率波動單獨抽出來，這樣剩下的才是「真實的人/薪/獎金」變動。
- **實發RMB差** 再細拆：底薪、獎金分紅、加班值班、補發補助、扣款。
- **本月用上月匯率重算** 勾選框 = 你目前手動做法（用上月匯率假設）。
- 各事業群／事業部下鑽，差額拆成「匯率」與「真實」兩塊。
- 新進／離職名單。

## 計算根據
每人實發PHP = `實發RMB × 匯率 + 小計PESO − 扣款小計PESO`
（已驗證對每個人都精準命中原始檔的「实发工资PHP比索」，整月瀑布殘差為 0）

## 部署
純靜態，無後端。三個檔：`index.html` `style.css` `app.js`，外加 SheetJS（CDN）。
丟到 GitHub Pages 即可。
